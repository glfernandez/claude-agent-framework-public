# Triage Labels

Maps the five canonical triage states to GitHub label strings.

| Canonical role    | GitHub label string | Meaning |
|-------------------|---------------------|---------|
| `needs-triage`    | `needs-triage`      | Maintainer needs to evaluate |
| `needs-info`      | `needs-info`        | Waiting on reporter for more information |
| `ready-for-agent` | `ready-for-agent`   | Fully specified, AFK-ready — an agent can implement with no human in the loop |
| `ready-for-human` | `ready-for-human`   | Needs human implementation (judgment, design, external access) |
| `wontfix`         | `wontfix`           | Will not be actioned |

Category labels:

| Canonical role | GitHub label string |
|----------------|---------------------|
| `bug`          | `bug`               |
| `enhancement`  | `enhancement`       |

Sprint label: update per sprint (e.g. `sprint-1`).

## Notes

- `ready-for-agent` means an AFK agent can implement with no human in the loop. Only apply when the issue has a full agent brief: context, acceptance criteria, seams to test, and an explicit out-of-scope list.
- Every AI-generated issue comment must start with: `> *This was generated by AI during triage.*`

## Bootstrap — create labels on a new repo

```bash
gh label create "needs-triage"    --color "ededed" --description "Maintainer needs to evaluate"
gh label create "needs-info"      --color "d4c5f9" --description "Waiting on reporter"
gh label create "ready-for-agent" --color "0075ca" --description "Fully specified, AFK-ready"
gh label create "ready-for-human" --color "e4e669" --description "Needs human implementation"
gh label create "wontfix"         --color "ffffff" --description "Will not be actioned"
gh label create "bug"             --color "d73a4a" --description "Something is broken"
gh label create "enhancement"     --color "a2eeef" --description "New feature or improvement"
gh label create "sprint-1"        --color "7057ff" --description "Sprint 1 scope"
```
