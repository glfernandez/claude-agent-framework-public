# Issue Tracker

Issues live in **GitHub Issues** on this project's repo.

## Tool

Use the `gh` CLI for all issue operations:

```bash
gh issue create --title "..." --body "..." --label "..."
gh issue list --label "needs-triage"
gh issue edit 42 --add-label "ready-for-agent" --remove-label "needs-triage"
gh issue comment 42 --body "..."
gh issue close 42
```

## Conventions

- Every AI-generated comment must start with: `> *This was generated by AI during triage.*`
- Apply both a **category** label (`bug` or `enhancement`) and a **state** label (see triage-labels.md) to every triaged issue.
- Sprint tracking: issues are grouped by milestone. Update `Current milestone` below each sprint.
- Backlog items should be filed as GitHub issues before implementation begins.

**Current milestone:** `Sprint 1`

## Issue body format (for agent-ready issues)

Every `ready-for-agent` issue must include:

```
## Context
[Why this issue exists. Link to PRD.]

## Acceptance criteria
- [ ] ...

## Seams to test
[Files, functions, or API routes that must be verified.]

## Out of scope
[Explicit list of what this issue does NOT cover.]

## Agent brief
[Any special constraints, dependency on upstream issues, or PARKED blockers.]
```
